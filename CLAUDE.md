## Что это

Сайт студии разработки и автоматизации SLK-labs. Сайт = главное доказательство компетенции, поэтому он
сам должен быть «вау», а не рассказывать про «вау».

**Концепция:** живая система (хребет) + дисциплина минимализма (форма). Сайт ведёт себя как система,
которая сама себя собирает — узлы и связи дышат, тянутся к курсору, замирают в покое. Это метафора
продукта: автоматизация = процессы, что живут и работают сами.

> **v7 (идея A из дорожной карты): живая система — хребет ВСЕГО сайта.** Сеть вынесена из героя в
> фиксированный canvas за всем контентом; страница = тёмное «живое поле», контент плывёт поверх в
> стеклянных «приборах». Сеть **рекоммутируется по скроллу** (формация/плотность/интенсивность под
> каждую секцию). Один WebGL-момент сохранён (один canvas). Раньше «живым» был только герой — теперь
> вся страница, но на контентных сценах сеть уходит на второй план (`dim`), чтобы не спорить с текстом.
> Детали — `ARCHITECTURE.md` §«Живая система (хребет)». Это сознательный отход v7-песочницы от
> «один живой момент» ради максимальной техничности (см. дорожную карту: v8 — лаборатория «строим
> вживую», v9 — работы-созвездие, v10 — рентген «под капотом»).

**Вербальный пакет (готов, использовать дословно):**
- Имя: **SLK-labs** (`SLK` — гротеск, `labs` — моноширинный суффикс).
- Дескриптор: «Сайты. Боты. Автоматизация.»
- Пэйофф (герой + финал манифеста): «Процессы, которые живут без вас.»
- Текст манифеста — целиком в `SLK-labs_Design_Foundation.md` §2 (готов под кинетику).

**Антидефолт (жёстко):** НЕ «крем + серриф + терракота», НЕ «чёрный фон + кислотный акцент».
Наша система = **светлая костяная база + гротеск + моно-слой + сигнальный зелёный только при взаимодействии**.

## Дизайн-система

Все значения — в `styles/tokens.css` (**единственный источник правды**). Tailwind `@theme` и `lib/motion.ts`
ссылаются на те же значения — вторую правду не плодить. Полные таблицы и готовые блоки — в `SLK-labs_Design_Tokens.md`.

### ⏳ Ждёт финальных значений из Claude Design
Подставить, когда придут (одной заменой в `tokens.css`):
- точный костяной фон (старт `#F2F0E9`),
- точный оттенок сигнального зелёного (старт `#00E08A`),
- файлы пары шрифтов (старт: **Geist + Geist Mono** в `public/fonts/`).

До замены работаем на стартовых значениях — каркас, лейаут и логика от этого не зависят.

### Контраст-правило (ЗАКОН, не нарушать)
Яркий зелёный `#00E08A` на костяном даёт **1.5:1** — текст/тонкие линии не читаются.
- зелёный = **только** против ink/тёмного (10.3:1) или как излучаемый свет/заливка/вспышка при взаимодействии;
- зелёный текст/линии на светлом → переключать на `--color-signal-ink` `#0B7A4B` (4.7:1);
- смысловые границы на светлом — графит/ink-opacity, **не** зелёный; `--color-hairline` — только декор.

### Моторика
Одна грамматика на весь сайт: контент **собирается и связывается** (не «фейдится»), линии дорисовываются,
разделы сшиваются нитью по скроллу, ховер зажигает узел. Сигнатурный easing — `expo.out` («оседание»).
Ревилы быстро оседают в покой. `prefers-reduced-motion` → анимации выключены, живая сеть → статичная структура.

## Карта реализации (что от чего зависит)

**Поток данных (однонаправленный):**
`lib/site.ts` + `lib/content/*` + `lib/cases.ts`  →  секции/Hero/Header  →  `app/page.tsx`  →  `app/layout.tsx`.
Компоненты НЕ содержат строк копирайта — только верстку. Текст меняется в `lib/`.

**Цепочки зависимостей:**
- `styles/tokens.css`  →  (Tailwind `@theme` генерит утилиты `bg-bone`/`text-ink`/`text-h2`/`ease-out-expo`… + CSS-vars `var(--…)`)  →  используется во ВСЕХ компонентах. Вторая правда запрещена.
- `app/layout.tsx`  →  `SmoothScroll` (Lenis) + `Header` + метаданные из `lib/site`.
- `SmoothScroll` / `Reveal` / `KineticText`  →  `lib/gsap` (плагины, `prefersReduced`) + `lib/motion` (easing/длительности). CSS-easing ↔ GSAP CustomEase — паритет.
- `app/layout.tsx`  →  `system/LiveSystem` (fixed-слой за ВСЕЙ страницей, не в герое)  →  `useSceneDriver()`
  (драйвер сцен для сети и хедера) + `SystemField` (dynamic ssr:false, r3f Canvas; монтаж отложен через
  requestIdleCallback)  →  `network/{config, shaders, simulation}`.
  Сеть слушает курсор на **window** (канвас `pointer-events:none`) — поэтому скролл и клики по CTA свободны.

**ГДЕ ЧТО КРУТИТЬ (чтобы не метаться по коду):**

| Хочешь поменять | Файл |
|---|---|
| Цвет / типографику / отступы / радиусы / easing-токены | `styles/tokens.css` |
| Текст героя/услуг/манифеста/контакта | `lib/content/*` |
| Кейсы (карточки в Works) | `lib/cases.ts` |
| Навигацию / SEO-метаданные / контакты | `lib/site.ts` |
| **Базовую физику/палитру живой сети** (тяга, чистая зона, радиус, скорости рекоммутации) | `components/system/network/config.ts` |
| **Поведение сети под секцией** (формация, плотность, дыхание, интенсивность `dim`) | `lib/scenes.ts` |
| Какая секция = какая сцена | `data-scene` на `<section>` + `lib/scenes.ts` |
| Тёмную подложку / стекло «приборов» | `styles/tokens.css` (`--color-substrate`, `*-glass-*`) + `.instrument/.tile-*` в `globals.css` |
| Длительности/easing анимаций (GSAP) | `lib/motion.ts` |
| Плавность скролла (инерция) | `components/SmoothScroll.tsx` (Lenis `lerp`) |
| Вид хедера на тёмной/светлой сцене | `components/Header.tsx` (берёт `surface` из `scene-store`) |
| Грамматику ревилов | `components/Reveal.tsx` + `html.js [data-reveal]` в `globals.css` |

**Инварианты (не нарушать):**
- Контраст-закон: `signal #00E08A` только против тёмного/как свет; на светлом — `signal-ink #0B7A4B`.
- `prefers-reduced-motion`: Lenis off, ревилы off (CSS), сеть → статичная структура (`simulation` reduced-ветка).
- Сеть — `pointer-events:none -z-10`, курсор на window. Не вешать r3f-события на канвас (сломает скролл/клики).
- Один WebGL-canvas на весь сайт (v7: фиксированный за контентом, не per-section), ленивый. tokens.css — единственный источник правды.
- Контент отделён от логики и в живой системе: `lib/scenes.ts` — ДАННЫЕ, `simulation.ts` — физика, `SystemField` — рендер. Не смешивать.
- Активная сцена — только через `scene-store` (его читают и сеть, и хедер). Второй источник «где страница» не плодить.
- Живая сеть = **three/r3f** (по локу, без drei). tsParticles НЕ использовать (был откатан).

## Гард-рейлы (проверять на каждом шаге)

- **Контраст-правило** из раздела выше — закон.
- **Производительность:** один WebGL-момент, lazy-load тяжёлого, цель — зелёные Core Web Vitals; тест на телефоне и среднем железе (почти половина сайтов не проходит CWV ещё до анимации).
- **Reduced-motion** уважается везде; сеть → статичная структура.
- **Мобайл — отдельный проход:** нет ховера, тач, упрощённая сеть; сделать так же тщательно.
- **A11y:** видимый focus, семантика, alt-тексты, контраст текста ≥ AA.
- **Ритм «тишина → удар»:** не анимировать всё подряд — это читается как AI-генерёжка; боль в одном месте (сеть), вокруг тихо.
- **Один источник правды** для токенов; CSS и GSAP двигаются одинаково.
- **Копирайт интерфейса:** имена — по тому, чем пользователь управляет, активный залог («Отправить», не «Сабмит»), пустые/ошибочные состояния — как направление к действию.
