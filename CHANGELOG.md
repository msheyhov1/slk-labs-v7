# Changelog

## v7.0.0 — 2026-06-29

Седьмая редакция (**slk-labs-v7**) — отдельный репозиторий. Идея A из дорожной карты:
**живая система как хребет всего сайта**. Живая сеть перестаёт быть «эффектом в герое» и
становится нервной системой всей страницы, которая **рекоммутируется по скроллу**.

### Что изменилось относительно v6
- **Сеть вынесена за весь сайт.** Фиксированный `Canvas` (`components/system/LiveSystem.tsx`)
  лежит за всем контентом; страница = тёмное «живое поле» (`--color-substrate`), контент плывёт
  поверх в **стеклянных «приборах»** (`.instrument` / `.tile-bone` / `.tile-ink`, полупрозрачные +
  `backdrop-blur` → сеть просвечивает). Один WebGL-момент на весь сайт сохранён (один canvas).
- **Рекоммутация по сценам.** Каждая секция объявляет `data-scene`; драйвер (`useSceneDriver`)
  по близости центра секции к центру вьюпорта пишет активную сцену в стор (`lib/scene-store.ts`).
  Сеть плавно перестраивает **формацию** узлов под секцию (`lib/scenes.ts`):
  герой — облако · работы — 3 созвездия · услуги — 3 колонны · манифест — разрежённая решётка ·
  контакт — схождение к одному фокусу. Плюс меняются плотность, дыхание и интенсивность (`dim`).
  На смене сцены — **импульс света** («система пересобралась»).
- **Палитра под тёмную подложку.** Рендер переведён на `NormalBlending` (корректно и на тёмном, и
  под стеклом); линии получили собственный шейдер с пер-сегментной альфой, узлы — атрибут видимости.
  Сеть всегда светлая/зелёная (видна на тёмном); контраст-закон соблюдён: текст — на костяном стекле,
  зелёный — свет/вспышка на тёмном.
- **Хедер — по сцене, не по порогу скролла.** Берёт `surface` активной сцены из того же стора:
  тёмная сцена → прозрачный + светлый текст; светлая (костяная панель) → стекло + тёмный.
- **Движок переехал** `components/hero/network/*` → `components/system/network/{config,shaders,simulation}`.
- `themeColor` → тёмная подложка `#0E1114`; `basePath`/`assetPrefix` экспорта → `/slk-labs-v7`.

### Инварианты сохранены
Контент отделён от логики (`lib/scenes.ts` — данные, `simulation.ts` — физика, `SystemField` — рендер);
один источник правды токенов; `prefers-reduced-motion` → статичная структура; тач — без тяги (тап = волна);
плотность сети ограничена (десктоп/мобайл cap). `npm run build` и экспорт `GH_PAGES` — зелёные.

## v6.0.0 — 2026-06-28

Шестая редакция (**slk-labs-v6**) — отдельный репозиторий на базе **v1**. Стиль, лейаут,
дизайн-система и живая сеть — как в v1; меняются только надписи (короткий копирайт) и поведение
клика по сети.

### Что изменилось относительно v1
- **Клик по живой сети больше не разбрасывает узлы.** Вместо удара (`kick`-импульс) клик пускает
  **волну света**: расширяющееся кольцо поднимает жар узлов на фронте и зажигает связи, через
  которые проходит. Силы не применяется — сеть стоит на месте, бежит только свет.
  Ручки — `components/hero/network/config.ts` (`wave.*`); логика — `simulation.ts` (`Wave` вместо `Shock`).
- **Короткий копирайт** (`lib/content/*`, `lib/cases.ts`): hero-спеки (`Сайты · Боты · Авто` …),
  Работы → «Запущенные системы» + «Несколько систем, которые работают сами.», Услуги → «Что собираем»,
  кейсы `K-01 New Link Food · K-02 Смена · K-03 Дозор`, Контакт → «Хочу такой же» +
  «Расскажите задачу — соберём систему под неё.», почта `hello@slk-labs.studio`.
- `SectionHead` получил необязательный `lead` (подзаголовок под заголовком секции).
- `basePath`/`assetPrefix` экспорта → `/slk-labs-v6`.

## v1.0.0 — 2026-06-26

Первая версия сайта **SLK-labs** (студия разработки и автоматизации).

### Что внутри
- Лендинг: `hero` (живая WebGL-сеть) / `works` / `services` / `manifesto` / `contact`.
- **Живая сеть** на three/@react-three-fiber: мягкая пружинная физика — узлы тянутся к курсору,
  «чистая зона» (не слипаются), импульс-волна по клику; в покое монохром, зелёный лишь при
  взаимодействии. `prefers-reduced-motion` → статичная структура.
- **Адаптивный хедер**: прозрачный + светлый на тёмном герое → костяной + тёмный при скролле.
  Якоря не уезжают под фикс-хедер (`scroll-padding-top`).
- Плавный скролл Lenis, ревилы GSAP/ScrollTrigger, кинетический манифест SplitType.
- Дизайн-система: `styles/tokens.css` → Tailwind v4 `@theme` (единый источник правды);
  шрифты Geist/Geist Mono (latin+cyrillic).
- A11y: семантика, skip-link, видимый focus, `reduced-motion` (включая CSS-переходы),
  тап-зоны ~44px; контраст-закон зелёного соблюдён.

### Архитектура
- Контент отделён от логики: `lib/site.ts`, `lib/content/*`, `lib/cases.ts`.
- Живая сеть разнесена: `components/hero/network/{config,shaders,simulation,LivingNetwork}`.
- Карта связей и тюнинга: `CLAUDE.md` (раздел «Карта реализации») + `ARCHITECTURE.md`.

### Стек (заперт)
Next.js 16 (App Router) · TypeScript · Tailwind v4 · GSAP + ScrollTrigger + CustomEase · Lenis ·
SplitType · three + @react-three/fiber + @react-three/drei.

### Дальше (не в v1)
- Фаза 4 — AI-ассистент (`@anthropic-ai/sdk`, серверный route handler).
- Фаза 5 — реальные кейсы, страницы `work/[slug]`, деплой на Vercel (выполняет владелец).
- ⏳ Финальные токены (костяной/зелёный/шрифты) — одной заменой в `styles/tokens.css`.
